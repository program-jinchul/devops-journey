## AWS 서비스 관리 클래스 설계

### 배운 것

**객체 지향 설계 (Class)**
- `AwsManager` 클래스로 EC2, RDS, S3 서비스를 하나의 객체로 통합 관리
- `__init__` : `boto3` 클라이언트를 사용하여 AWS 서비스 객체 초기화

**코드 구조 개선**
- **기능 분리** : 서비스별 제어 로직을 각각의 메서드로 분리하여 유지보수성 향상
- **매개변수 전달** : `input()`으로 받은 `action` 값을 메서드 인자로 전달하여 유연하게 기능 제어
- **실행 흐름** : 클래스 정의는 상단에, 실제 `input` 및 조건문은 클래스 밖(하단)에 배치하여 구조 명확화

### 코드 작성 시 주의사항
- `self.` 사용 : 클래스 내부 메서드에서 `boto3` 클라이언트를 호출할 때 필수
- 들여쓰기(Indentation) : `if-elif` 실행부를 클래스 블록 밖으로 분리하여 객체 생성 후 실행되도록 설정
- 인자 매칭 : 메서드 정의부의 `action`과 호출 시 전달하는 변수를 정확히 일치

### 문제 풀이
- `AwsManager` 클래스 생성 및 `boto3` 클라이언트 구성
- 서비스(`ec2`, `rds`, `s3`)별 `start`/`stop` 로직 분기 처리
- `input()`을 통한 동적 서비스 제어 기능 구현
